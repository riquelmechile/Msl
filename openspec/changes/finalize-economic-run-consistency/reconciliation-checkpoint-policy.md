# Reconciliation, Lease, Fence, and Checkpoint Policy

No HTTP runs inside SQLite transactions. Admission records the open fence generation/token digest, database generation, observed epoch, and lease owner. Fence default is TTL 90s, renew every 30s, recover every 15s; every acquisition/renewal/release is owner/token-digest/generation CAS. Lease acquire, renew, release, backlog mutation, and every migration writer validate the same open fence. The final `BEGIN IMMEDIATE` rechecks: matching open fence generation/token, database generation, matching unexpired lease owner/token, and no restore state. It repeats both checks immediately before commit. Any miss rolls back all final economic writes and is a global abort.

Checkpoint CAS remains at most three strictly-greater expected-version attempts: `advanced`, `already-applied`, `stale`, `concurrent`, `missing`, or `retry-exhausted`. `missing` and `retry-exhausted` roll back the final transaction.

R2's future operational-failure migration, not 1010, will require an idempotent `run_failure_intent` for the same `run_id` before the main transaction. After rollback, a **separate** short operational transaction will CAS-update that intent and persist failed/aborted terminal detail, CAS classification and expected/observed version, source health, and deduplicated alert intent. It will bind the original admission fence generation/token digest, database generation, observed epoch, and lease owner; a stale writer or zero-row CAS is rejected. It validates fence state but does not require the lost lease and does not advance checkpoints. Startup recovery resumes incomplete intent states idempotently for that same run ID. If it fails, the caller fails closed, returns sanitized `operational-persistence-blocked`, marks readiness unavailable, and attempts an independent allowlisted emergency alert. It never reports completion.

The successful final transaction writes outcome/backlog, evidence, run, health, checkpoint, alert intent, then increments `write_epoch` once. Fence checks and lease operations never increment the epoch.
