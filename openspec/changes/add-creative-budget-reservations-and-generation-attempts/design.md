# Design: Creative Budget Reservations and Generation Attempts

## Architecture

SQLite is sole authority for micros, reservations, attempts, and recovery. Flow: `claim -> reserve -> prepare+lease -> dispatch -> atomic attempt/reservation reconcile -> settle`. Legacy USD is derived from micros, never read for admission/reconciliation/reporting totals.

| Decision | Rationale |
|---|---|
| `BEGIN IMMEDIATE`; 5s busy timeout | Serializes cap/state changes; rollback leaves no partial authority. |
| 90s fenced lease; renew/30s | Enables takeover without releasing possibly-sent work. |
| Trusted definitely-unsent evidence only | POST is non-atomic; uncertainty must not duplicate charges. |

## Canonical Storage

Creative v4 owns exactly:

```sql

```

`up` uses `IF NOT EXISTS`; `isApplied` compares normalized `sqlite_master.sql` and all table/FK/index PRAGMAs to this definition, including CHECKs/predicates. Partial/foreign objects block startup; `foreign_key_check` must be empty.

## Contracts and Fencing

```ts

type Conflict={kind:"identity-mismatch";fields:readonly string[]}|{kind:"divergent-repeat";field:string}|{kind:"invalid-state";state:string;expected:readonly string[]}|{kind:"amount-over-reserved";reserved:Micros;actual:Micros}|{kind:"busy";retryable:true}|{kind:"lease-lost";attemptId:string;generation:number};type Result<T>={ok:true;value:T;idempotent:boolean}|{ok:false;conflict:Conflict};
type NoSubmissionProof=Readonly<{kind:"transport-before-send";authority:"minimax-transport";bodyBytesOffered:0;evidenceRef:string}|{kind:"provider-rejection";authority:"minimax-adapter";accepted:false;charged:false;providerRequestId:string;evidenceRef:string}|{kind:"operator-reconciliation";authority:"creative-ops";confirmedUnsent:true;confirmedUncharged:true;evidenceRef:string}>;
type ReservationStore={reserve(i:Key&{currency:string;utcDay:string;requested:Micros;dailyCap:Micros;jobCap:Micros;expiresAt:number;now:number}):Result<Reservation>;get(k:Key):Reservation|undefined;commit(i:Key&{actual:Micros;evidenceRef:string;now:number}):Result<Reservation>;release(i:Key&{proof:NoSubmissionProof;now:number}):Result<Reservation>;renewHold(i:Key&{fence:Fence;expiresAt:number}):Result<Reservation>;expireDue(i:{now:number;limit:number}):Result<readonly Reservation[]>};
type AttemptStore={prepare(i:Key&{messageId:string;provider:string;model:string;idempotencyKey:string;requestHash:string;estimated:Micros;requestEvidence:Evidence;leaseOwnerId:string;leaseExpiresAt:number;now:number}):Result<LeaseGrant>;get(attemptId:string):Attempt|undefined;acquireDue(i:{ownerId:string;now:number;limit:number}):Result<readonly LeaseGrant[]>;renewLease(i:Fence&{expiresAt:number}):Result<Attempt>;markDispatching(i:Fence):Result<Attempt>;markSubmitted(i:Fence&{taskId:string;providerRequestId:string|null;evidence:Evidence}):Result<Attempt>;markAmbiguous(i:Fence&{evidence:Evidence}):Result<Attempt>;complete(i:Fence&{actual:Micros;evidence:Evidence}):Result<Attempt>;fail(i:Fence&{error:Evidence;proof:NoSubmissionProof}):Result<Attempt>};
```

Ownership proof is the exact `Key`; every transition predicates all four columns. Tokens are random 32-byte values; DB stores SHA-256. Acquire after expiry increments generation and replaces owner/digest/expiry atomically; old fences fail. Dispatch/reconcile require matching tuple and unexpired lease.

`expireDue` may expire only overdue `held` rows for which no exact-key attempt exists. It NEVER releases/expires a hold linked to `prepared`, `dispatching`/sending, `submitted`, `ambiguous`, terminal, or actively reconciled work because of lease expiry. Takeover continues the same attempt; possibly-sent holds remain committable after reservation/lease timeouts. Release requires `NoSubmissionProof` from its named authority, or terminal same-attempt provider/operator evidence. Unproven 429/5xx/network loss/abort/timeout becomes `ambiguous`; never re-POST.

Completion/failure is one `BEGIN IMMEDIATE`: fence/key validation, evidence, commit/release, attempt close. Exact retries are idempotent; divergent closure conflicts. Settlement follows commit.

## Migration, Workers, Tests

`agentMessageBusStore.ts` exports `registerAgentMessageBusMigrations(registry)` for current v1-v3 and `runLegacyAgentMessageBusMigrations(db)`; each v1-v3 step gets exact `isApplied` ownership proof. `createAgentMessageBusStore(db,{migrate=true})` defaults to unchanged current behavior. Persistence opens/pragmas, registers bus v1-v3 then creative v4 in one registry when enabled, applies, proves bus ownership then v4/FKs, and constructs bus with `{migrate:false}`. Disabled mode runs exported legacy bus orchestration, proves v1-v3, then v4 `up`/proof. Readiness becomes `ready` only afterward; otherwise `blocked`. Durability enabled fails closed; legacy `CostLedger` is allowed only before persisted activation.

Polling remains immediate/non-overlapping at `MSL_CREATIVE_STUDIO_POLL_INTERVAL_MS` (30s); global 15-minute scheduler excludes Creative. Recovery runs only when ready, renews attempt/hold during queries, and drains/releases leases before DB close. `runCreativeDurabilityRollback({operatorEvidenceRef,now})`/rollback script drain and refuse disable with ambiguous/leased holds. Metadata retains `attemptId,reservationId,estimatedCostMicros,actualCostMicros,provider,model,promptSha256,referenceSha256s,requestedByAgent,channel,jobId`; USD derives from micros. Runtime owns busy >1%/5m, recovery >=3 cycles, ambiguity >15m, hold >30m; Finance/Creative Ops owns nonzero drift.
