# creative-image-provider-contracts Specification

## Purpose

`ImageProvider` port and adapters for creative image generation. OpenAI `gpt-image-2` primary. BRIA feature-gated with `awaiting-provider-consent` state (entered from `running`, message retained `deferred`). MiniMax excluded — video only. Provider-agnostic ML format compliance.

## Requirements

### Requirement: ImageProvider Normalized Output with ML Format

`ImageProvider.generate(input)` SHALL return: image bytes/local URI, pixel dimensions, MIME format, SHA-256 of prompt, every canonical reference SHA-256, every output SHA-256, provider, model, `estimatedCostUsd`, `actualCostUsd`, moderation/safety flags, policy flags, audit linkage (`jobId`, timestamp). Product images for MercadoLibre channel SHALL be normalized/validated to 1200×1200 px, JPEG, RGB before `needs-human-review`. Non-ML channels MAY vary per channel constraint. Every normalized output, queue result, persistent audit entry, and Cortex evidence SHALL carry separate `estimatedCostUsd` and `actualCostUsd` fields; no generic `cost` replacement.

#### Scenario: ML product image

- GIVEN channel `mercadolibre`; product kind
- WHEN image generated by any ImageProvider
- THEN normalized to 1200×1200, JPEG, RGB before review

#### Scenario: Non-ML channel

- GIVEN channel `storefront`
- WHEN image generated
- THEN resolution per channel constraint

#### Scenario: Output hash mismatch

- GIVEN downloaded bytes differ from provider-returned hash
- WHEN validated
- THEN flagged in audit

### Requirement: Reference Hash Gate and Prompt Safety

Every canonical reference hash SHALL be present and non-null before provider dispatch. Missing or mismatched hash SHALL block the call and record evidence; SHALL NOT normalize missing as acceptable. ALL providers SHALL receive structured tiered prompts: identity and reference context first, then allowlisted hints. SHALL NOT include raw seller title, free text, or untrusted user input. Provider optimizer rewriting SHALL be disabled.

#### Scenario: Reference present

- GIVEN all reference hashes available
- WHEN pre-dispatch check runs
- THEN provider call proceeds

#### Scenario: Reference missing

- GIVEN one or more reference hashes unavailable
- WHEN pre-dispatch check runs
- THEN call blocked; audit recorded; job → `failed`

#### Scenario: Prompt safety

- GIVEN seller title present in raw input
- WHEN prompt assembled for provider
- THEN structured format: identity/references + hints; raw title excluded

#### Scenario: Optimizer disabled

- GIVEN prompt sent to provider
- WHEN provider call made
- THEN `prompt_optimizer: false` or equivalent flag set

### Requirement: OpenAI Primary Provider

`OpenAIImageProvider` SHALL be primary using `gpt-image-2`. SHALL send reference-conditioned prompts for product images per structured prompt contract. SHALL NOT silently delegate to another provider. Budget SHALL be checked via `DurableCostLedger.canAfford()` before every call.

#### Scenario: Product image generated

- GIVEN valid reference, structured prompt, budget ok
- WHEN OpenAI called
- THEN `gpt-image-2` invoked; normalized output returned

#### Scenario: Budget check fails

- GIVEN `canAfford()` returns false
- WHEN before OpenAI call
- THEN call blocked; evidence recorded

#### Scenario: OpenAI safety rejection

- GIVEN OpenAI returns content policy rejection
- WHEN provider returns
- THEN BRIA SHALL NOT be tried

### Requirement: BRIA Deterministic Consent Fallback

OpenAI fails with allowed non-safety failure from `running` state; BRIA feature gate enabled; no existing consent → `running → awaiting-provider-consent`; same bus message retained in durable nonterminal `deferred`; ZERO new messages; ZERO BRIA calls; ZERO asset transfers; reason recorded. Consent within 24h: re-check seller identity, policy compliance, BRIA enablement, and budget. If budget allows → `running` with same message; one BRIA call. If budget insufficient: consent remains recorded; job transitions to `awaiting-budget-approval` (phase=runtime); same message retained `deferred`; SHALL NEVER return to `awaiting-provider-consent`. Refusal or no-consent within 24h → `failed`; terminal bus fail once. OpenAI safety/content rejection SHALL NEVER enter consent fallback. Audit SHALL be recorded.

#### Scenario: Fallback entered, no consent

- GIVEN OpenAI 5xx from `running`; BRIA enabled; no existing consent
- WHEN fallback evaluated
- THEN `running → awaiting-provider-consent`; same msg `deferred`; zero BRIA calls; zero assets

#### Scenario: Consent granted, re-checks ok

- GIVEN consent within 24h; seller/policy/budget ok
- WHEN BRIA called
- THEN → `running` same message; one BRIA call; audited

#### Scenario: Consent granted, budget insufficient

- GIVEN consent present; `canAfford()` returns false
- WHEN pre-BRIA budget check
- THEN consent retained; → `awaiting-budget-approval` (runtime); same msg `deferred`; never returns to consent state

#### Scenario: Refusal or 24h timeout

- GIVEN CEO refuses or 24h elapsed without consent
- WHEN fallback ends
- THEN → `failed`; zero BRIA calls; terminal bus fail once

#### Scenario: Safety rejection blocks consent

- GIVEN OpenAI returns content safety rejection
- WHEN fallback evaluated
- THEN never enters `awaiting-provider-consent`; terminates per policy

### Requirement: No Silent Dual Calls

Primary provider success → other provider SHALL NOT be called. Primary non-safety failure → fallback per consent gates. Safety/content rejection → no fallback.

#### Scenario: Primary succeeds

- GIVEN OpenAI returns valid image
- WHEN job completes
- THEN BRIA never called

#### Scenario: Primary fails, consent absent

- GIVEN OpenAI 5xx from `running`; BRIA enabled; no consent
- WHEN fallback evaluated
- THEN `running → awaiting-provider-consent`; BRIA not called while waiting; same message retained
