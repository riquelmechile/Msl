# Delta for specialist-daemons

## MODIFIED Requirements

### Requirement: creativeStudioDaemon

`creativeStudioDaemon` MUST export `investigate(claim: AgentMessage): Promise<DaemonResult>`. It SHALL poll messages where `receiverAgentId = "creative-studio"` and `status = "pending"`, claim them, and route by `CreativeJobKind`: product image jobs (`product-cover-i2i`, `product-gallery-i2i`, `social-pack`) to OpenAI primary with BRIA consent-gated fallback via `ImageProvider` port; video jobs (`ml-clip-vertical`, `ml-clip-vertical-30s`) to MiniMax video-only. It SHALL persist outputs locally, run ML pre-diagnosis for `mercadolibre` channel image jobs, and respond with `CreativeExecutionResult`. It SHALL enforce budget via `DurableCostLedger.canAfford()` before every paid generation call. It SHALL be disabled when `MSL_CREATIVE_STUDIO_ENABLED` is not `"true"`, returning empty findings.
(Previously: all jobs routed to MiniMax for both image and video generation; no provider selection or ImageProvider port.)

#### Scenario: Image job received

- GIVEN bus message with `kind: "product-cover-i2i"`
- WHEN daemon processes
- THEN OpenAI primary called via ImageProvider; BRIA fallback per consent; asset persisted; result returned

#### Scenario: Video job received

- GIVEN bus message with `kind: "ml-clip-vertical"`
- WHEN daemon processes
- THEN MiniMax called for video only via VideoTaskStore; async polling; asset persisted

#### Scenario: Budget exceeded

- GIVEN job cost > remaining daily budget
- WHEN daemon validates
- THEN job enters `awaiting-budget-approval` per phase rules; message retained if runtime

#### Scenario: Env gate disabled

- GIVEN `MSL_CREATIVE_STUDIO_ENABLED=false`
- WHEN daemon cycle starts
- THEN empty findings; no bus polling

## ADDED Requirements

### Requirement: Provider Ownership Routing

Provider ownership for creative asset generation SHALL be: product images route to OpenAI primary with BRIA consent-gated fallback via `ImageProvider` port; MiniMax routes video jobs only. ML image format/compliance (1200×1200, JPEG, RGB) SHALL remain provider-agnostic and enforced by `ImageProvider` adapters. MiniMax capability purpose SHALL become video-only after archive.

#### Scenario: Product cover image → OpenAI

- GIVEN channel `mercadolibre`; kind `product-cover-i2i`; budget ok
- WHEN daemon routes
- THEN OpenAI `gpt-image-2` called via ImageProvider

#### Scenario: OpenAI allowed failure → BRIA consent

- GIVEN OpenAI returns non-safety failure; BRIA enabled; no existing consent
- WHEN daemon evaluates fallback
- THEN job → `awaiting-provider-consent`; same message retained `deferred`

#### Scenario: Video job → MiniMax only

- GIVEN kind `ml-clip-vertical`
- WHEN daemon routes
- THEN MiniMax called for video; ImageProvider not invoked

#### Scenario: ML format provider-agnostic

- GIVEN image generated by any ImageProvider for channel `mercadolibre`
- WHEN output normalized
- THEN 1200×1200, JPEG, RGB enforced regardless of which provider generated it
